# 栖卷 (MyBookStore) - 扫码录入模块详细说明书

本文档是对当前应用内“扫码录入”模块（`ScanTab.ets`）所有功能、交互弹窗、API 请求规范以及数据格式的详细拆解说明。

---

## 一、 模块核心功能概览

“扫码录入”是本书城应用快速添加实体书籍的核心入口，融合了系统相机扫描、网络数据抓取、本地相册裁切以及纯手动录入的多种手段，保障用户在任何网络或数据状况下都能顺畅录入。

主要功能拆解如下：
1. **动态权限管理**：每次切入该 Tab 页，都会实时检查 `ohos.permission.CAMERA` 权限，若未授权则动态拉起系统授权弹窗。
2. **XComponent 实时预览**：基于 `@kit.ScanKit` 的 `customScan` 能力，将相机流绑定到全屏的 `XComponent` 上进行高性能实时解析。
3. **多模态扫码**：配置了 `scanCore.ScanType.ALL`，支持识别各类条形码（包括标准 ISBN 的 EAN-13 码）。
4. **异步网络解析**：扫码成功提取 ISBN 后，自动中断扫描，并在后台向用户配置的自定义 API 网关发起请求解析书籍元数据。
5. **智能封面下载**：网络 API 成功返回带有效 `cover` URL 的数据时，会通过 `http.createHttp` 将远程图片静默下载缓存至本地（防止后续图片服务器防盗链或失效），确保无网也能看封面。

## 二、 交互弹窗设计 (Dialogs)

为了保证体验的连贯性，该模块设计了三层降级交互弹窗：

### 1. 成功确认弹窗 (`showConfirmDialog`)
**触发条件**：API 服务器成功解析了 ISBN 并返回了正确的书籍 JSON 数据。
**弹窗内容**：
- 顶部显示：【扫码成功】
- 主体部分：展示书籍的精美封面、书名、作者、出版社。
- 底部操作：`取消` / `加入书架`。
点击加入后，书籍将被放入系统的全局书架库中，封面背景色会随机生成匹配。

### 2. 无 API / 接口失败提示弹窗 (`showNoApiDialog`)
**触发条件**：用户尚未配置 API，或者 API 接口崩溃、超时、返回 404 等。
**弹窗内容**：
- 顶部显示：【未配置 API / 找不到该书籍】
- 主体文本提示：“您尚未在设置中配置有效的书籍信息 API，或者接口未能查询到该书籍信息。您可以手动录入，或前往「我-设置」中配置接口。”
- 底部操作：`取消` / `手动录入`。

### 3. 手动录入全表单弹窗 (`showManualEntryDialog`)
**触发条件**：用户在上述提示弹窗中点击了“手动录入”，或在扫码主界面直接点击了右上角的加号按钮。
**弹窗内容**：
- 顶部显示：【手动录入书籍】
- **表单字段**：
  - **封面上传**：点击顶部占位图（调用 `picker.PhotoViewPicker` 拉起系统图库，选定后自动拷贝到私有沙箱 `filesDir` 中）。
  - **书名** (必填)
  - **作者** (必填)
  - **出版社**
  - **ISBN** (如果通过扫码失败进来，会自动带入扫出的那串数字)
  - **页数** (只能输入数字)
  - **内容简介** (支持多行输入框)
- 底部操作：`取消` / `保存`。

## 三、 自定义 API 配置规范与 Token 格式

由于主流书籍 API 存在各种限制，本应用采用了**极致开放**的架构，允许用户在【我 -> 偏好设置】中自行填写 API 网关。

### 1. 请求方法与 URL 拼接
发起请求采用 **GET** 方法。
引擎会读取用户填写的 `customApiUrl`。如果在 URL 中未探测到预设的插值逻辑，则默认采用 URL 参数追加的形式拼凑 ISBN，例如：
- 如果您填入的 API 为：`https://api.example.com/book`
- 系统会实际请求：`https://api.example.com/book?isbn=9787111111111`
- 如果您填入带有参数位的 URL（如 `http://test.com/api?action=get&isbn=`），系统会智能在其末尾直接追加，如 `http://test.com/api?action=get&isbn=978...`

### 2. Header & Token 鉴权格式
如果在设置中配置了 `customApiToken`，系统将在 HTTP 请求头（Headers）中加入以下字段来进行鉴权认证：
```http
Content-Type: application/json
X-API-Token: <您的Token内容>
```
*注：这里的键名为固定的 `X-API-Token`。*

## 四、 返回 JSON 数据契约 (Response Format)

目标 API 服务器响应 HTTP `200 OK` 状态码时，应返回一个 **扁平化的 JSON 对象**。系统提取的关键字段如下：

| JSON 字段名 | 类型 | 说明 | 必填 | 默认降级值 |
| :--- | :--- | :--- | :--- | :--- |
| **`title`** | String | 书籍的正标题 | 是 | "未知书名" |
| **`author`** | String | 书籍的作者/译者 | 是 | "未知作者" |
| **`cover`** | String | 封面高清图片绝对链接 (http/https 开头) | 否 | 缺省占位图 |
| **`summary`** | String | 内容简介、书籍大纲 | 否 | "暂无内容简介。" |
| **`publisher`**| String | 出版社名称 | 否 | "未知出版社" |
| **`pubdate`** | String | 出版年月 (通常为 YYYY-MM 格式) | 否 | "--" |
| **`pages`** | String / Number | 全书总页码 | 否 | `0` |

### 标准服务器返回示例

```json
{
  "title": "设计模式：可复用面向对象软件的基础",
  "author": "Erich Gamma / Richard Helm",
  "cover": "https://img3.doubanio.com/view/subject/l/public/s1011322.jpg",
  "summary": "本书结合设计实例从面向对象的设计中精选出23个设计模式...",
  "publisher": "机械工业出版社",
  "pubdate": "2000-09",
  "pages": 254
}
```

*如果您的服务器返回了多余的嵌套数据（如 `data: { title: ... }`），由于目前解析器是针对顶层平铺结构编写的，可能会导致提取失败。在架设中转 API 时，请务必洗出如上文所示的最外层扁平 JSON。*
