# 栖卷 (MyBookStore) - 个人中心模块详细说明书

本文档对当前应用内的“个人中心”模块（涵盖 `ProfileTab.ets` 以及系统设置面板 `SettingsOverlay.ets`）的设计架构、底层统计算法、Canvas 自定义绘图以及数据导出功能进行详细拆解说明。

---

## 一、 模块功能概览

个人中心（Profile）模块是全应用用户行为数据的集散中心。通过读取内存中的全局存储（`AppStorage`），实现各种维度的即时统计与可视化图表渲染。同时，它也是进入系统设置、调整隐藏 API Token 权限、执行数据导出的入口。

---

## 二、 全局概览统计板 (Top Stats)

位于界面顶部的横向数据总览卡片。核心指标全实时计算：

1. **总阅读时长 (分钟)**：遍历全局对象 `globalReadingSessions` 中的每一条流水记录，累加 `duration`（时长），并在前端格式化展示。
2. **总藏书量 (本)**：直接映射全局对象 `globalBooks` 数组的总长度。
3. **总笔记数 (条)**：直接映射全局对象 `globalNotes` 数组的总长度。

---

## 三、 数据可视化图表层 (Charts System)

个人中心内嵌了基于用户藏书属性的智能分析引擎，包含三个核心维度的统计图表：

### 1. 藏书阅读状态统计
- **算法**：扫描书籍列表的 `status` 字段。
- **状态分类**：
  - **在读 (Reading)**：橙色 `#D99B26`
  - **已读 (Read)**：主绿色 `#4E8975`
  - **未读 (Unread)**：灰绿色 `#8C9D96`
- **展现形式**：展示对应的本书与占比（%）。若没有对应状态的数据，则自动将其从视图中剔除。

### 2. 图书品类偏好分析 (Category Chart)
- **算法**：基于书籍的 `category` 字段计算分类词频。处理了未填写或空串的边缘情况，归一化为“未分类”。
- **聚合逻辑**：对所有出现过的分类进行倒序排列，**仅提取 Top 4 的核心分类**分配独立颜色，第 5 名及之后的分类统一卷入并求和归类为**“其他”**分类。
- **展现形式**：以不同色块的进度条/色段呈现用户的阅读涉猎偏好。

### 3. 本地构建的 Canvas 折线图 (入库时间趋势)
该模块彻底抛弃了重量级的第三方图表库，采用 HarmonyOS 原生的 `CanvasRenderingContext2D` 手绘平滑折线图，极具工业级优化价值。
- **动态滤镜**：支持切换 **“近一年 (Year)”** 和 **“全部 (All)”** 视图。
- **近一年视图算法**：向前推算 12 个月（格式化为 `YYYY-MM`），扫描书籍的 `addTime` 字段，按月度汇聚新增书籍数，并动态标注 Y 轴最高点（偶数归一化）。
- **全部视图算法**：采用**累计总额算法**，计算近 5 年来每年底的藏书总量，并将 5 年前的数据聚合为“更早”节点。
- **渲染细节**：实现了贝塞尔曲线连接/折线连接、X 轴文本标注、极细网格底纹，以及折线下方由实色渐变到透明的半透明填充遮罩 (`rgba(78, 137, 117, 0.08)`)。

## 四、 设置与系统工具弹窗 (Settings Overlay)

通过点击界面右上角的设置图标，拉起半模态的 `SettingsOverlay` 组件，提供深度控制和运维功能：

### 1. 隐藏的 API 认证网关 (邀请码)
- **机制**：在“邀请码”输入框中输入的值，会动态绑定到全局变量 `customApiToken`。
- **持久化**：使用 `@ohos.data.preferences` 写入沙箱文件（键名为 `customApiToken`），保证应用重启后凭据不丢失。供扫码网关鉴权使用（即之前说明书中的 `X-API-Token`）。
- **隐藏配置**：根据用户偏好，自定义 API 的完整 URL 配置地址被隐去了 UI 输入框，保持了前端界面的极简（底层依旧支持读取 `customApiUrl`）。

### 2. 纯本地化数据导出 (Export Books)
- **机制**：通过 `@ohos.file.picker` 拉起系统的 `DocumentViewPicker`，引导用户在系统文件管理器中创建一个名为 `栖卷_backup.txt` 的纯文本文件。
- **落盘逻辑**：调用原生 `fs.openSync` 模块，将全局 `books` 数据序列化为排版优良的 JSON 字符串（`JSON.stringify(books, null, 2)`）并写入。
- **价值**：实现了书籍数据的去中心化管理与备份迁移。

### 3. 设备 ID 与 辅助信息
- **剪贴板对接**：通过 `@ohos.pasteboard` 实现设备匿名 ID 的一键复制。
- **关于我们**：点击拉起独立的 `AboutOverlay`，展示应用开发团队与版权信息。
